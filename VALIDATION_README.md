# Система валидации данных

## Обзор

В проекте реализована комплексная система валидации данных, которая включает:

1. **Серверную валидацию** - проверка данных на стороне Django
2. **Клиентскую валидацию** - проверка данных в браузере
3. **Проверку дубликатов** - предотвращение создания дублирующихся записей
4. **Валидацию форматов** - проверка корректности IP-адресов, дат, числовых значений

## Файлы валидации

### 1. `netassets/assets/validators.py`
Содержит все функции валидации:

#### Валидация форматов:
- `validate_ip_address()` - проверка IP-адресов
- `validate_ip_range()` - проверка IP-диапазонов (CIDR и диапазоны)
- `validate_subnet_mask()` - проверка масок подсети
- `validate_inventory_number()` - проверка инвентарных номеров
- `validate_serial_number()` - проверка серийных номеров
- `validate_barcode()` - проверка штрихкодов
- `validate_vlan()` - проверка VLAN (1-4094)
- `validate_port()` - проверка портов (1-65535)

#### Проверка дубликатов:
- `check_ip_duplicate()` - проверка дубликатов IP-адресов
- `check_network_address_duplicate()` - проверка дубликатов сетевых адресов
- `check_inventory_number_duplicate()` - проверка дубликатов инвентарных номеров
- `check_serial_number_duplicate()` - проверка дубликатов серийных номеров
- `check_barcode_duplicate()` - проверка дубликатов штрихкодов
- `check_pc_name_duplicate()` - проверка дубликатов имен ПК
- `check_code_duplicate()` - проверка дубликатов кодов
- `check_printer_name_duplicate()` - проверка дубликатов имен принтеров

### 2. `netassets/assets/models.py`
Модели с интегрированными валидаторами:

- Добавлены валидаторы в поля моделей
- Реализованы методы `clean()` для проверки дубликатов
- Валидация выполняется при сохранении объектов

### 3. `netassets/assets/forms.py`
Формы с дополнительной валидацией:

- Каждая форма имеет метод `clean()` для дополнительной проверки
- Валидация длины полей
- Проверка обязательных полей
- Интеграция с валидаторами из `validators.py`

### 4. `netassets/assets/views.py`
Views с обработкой ошибок валидации:

- Обработка `ValidationError` и `IntegrityError`
- Отображение пользователю понятных сообщений об ошибках
- Валидация при импорте CSV

## Клиентская валидация

### JavaScript функции (в `base.html`):

```javascript
// Валидация IP-адреса
function validateIPAddress(ip) {
    const ipPattern = /^(?:(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)\.){3}(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)$/;
    return ipPattern.test(ip);
}

// Валидация IP-диапазона
function validateIPRange(range) {
    const rangePattern = /^(?:(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)\.){3}(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)(?:\/(?:[0-9]|[1-2][0-9]|3[0-2])|-(?:(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)\.){3}(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?))$/;
    return rangePattern.test(range);
}

// И другие функции валидации...
```

### Визуальная обратная связь:

- Поля с ошибками подсвечиваются красным
- Поля с корректными данными подсвечиваются зеленым
- Показываются сообщения об ошибках под полями
- Валидация происходит при потере фокуса (событие `blur`)

## Правила валидации

### IP-адреса:
- Формат: `192.168.1.1`
- Каждый октет от 0 до 255
- Поддержка IPv4

### IP-диапазоны:
- CIDR формат: `192.168.1.0/24`
- Диапазон: `192.168.1.1-192.168.1.254`

### Маски подсети:
- Формат: `255.255.255.0`
- Каждый октет от 0 до 255

### Инвентарные и серийные номера:
- Только заглавные буквы, цифры, дефисы и подчеркивания
- Регулярное выражение: `^[A-Z0-9\-_]+$`

### Штрихкоды:
- Только цифры
- Регулярное выражение: `^[0-9]+$`

### VLAN:
- Числовое значение от 1 до 4094

### Порты:
- Числовое значение от 1 до 65535

### Длина полей:
- Имя ПК: минимум 3 символа
- Имя принтера: минимум 3 символа
- Код устройства: минимум 2 символа
- Название сущности: минимум 2 символа
- Название поля: минимум 2 символа

## Проверка дубликатов

Система проверяет дубликаты для следующих полей:

1. **IP-адреса** - уникальны во всех таблицах
2. **Сетевые адреса** - уникальны в таблице компьютеров
3. **Инвентарные номера** - уникальны в таблице компьютеров
4. **Серийные номера** - уникальны в таблице компьютеров
5. **Штрихкоды** - уникальны в таблице компьютеров
6. **Имена ПК** - уникальны в таблице компьютеров
7. **Коды** - уникальны в таблице сетевых устройств
8. **Имена принтеров** - уникальны в таблице принтеров

## Обработка ошибок

### Серверная сторона:
```python
try:
    obj = form.save()
    messages.success(request, 'Объект успешно создан!')
except ValidationError as e:
    messages.error(request, f'Ошибка валидации: {e}')
except IntegrityError as e:
    messages.error(request, f'Ошибка целостности данных: {e}')
```

### Клиентская сторона:
```javascript
// Показ ошибки
function showValidationError(field, message) {
    field.classList.add('validation-error');
    // Показ сообщения об ошибке
}

// Показ успеха
function showValidationSuccess(field) {
    field.classList.add('validation-success');
    // Скрытие сообщения об ошибке
}
```

## Валидация при импорте CSV

При импорте CSV файлов:

1. Проверяется формат файла
2. Валидируется каждая строка
3. Проверяются дубликаты
4. Выводится подробный отчет об ошибках
5. Показывается количество успешно импортированных записей

## Использование

### В формах:
```html

        <div class="text-danger">{{ form.field_name.errors.0 }}</div>
```

### В моделях:
```python
def clean(self):
    super().clean()
    # Проверка дубликатов
    if self.ip_address:
        check_ip_duplicate(self.ip_address, NetworkDevice, self.id)
```

```python
if form.is_valid():
    try:
        obj = form.save()
        messages.success(request, 'Успешно!')
    except ValidationError as e:
        messages.error(request, f'Ошибка: {e}')
```

## Преимущества системы

1. **Двойная защита** - валидация на сервере и клиенте
2. **Понятные сообщения** - пользователь видит конкретные ошибки
3. **Предотвращение дубликатов** - автоматическая проверка уникальности
4. **Визуальная обратная связь** - цветовая индикация полей
5. **Мгновенная валидация** - проверка при вводе данных
6. **Безопасность** - серверная валидация как основная защита
